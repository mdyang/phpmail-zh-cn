信纸模板规范

 

1.       全局规范

 

a.       文件格式

信纸模板文件应该是标准的ZIP文件，通过系统的信纸模板管理模块上传，方可在撰写邮件时使用；

2.       目录结构

模板应该由以下文件和文件夹构成，如下图所示：

 

a.       main.html (文件)

此文件为模板的主文件，注意文件名一定要是main.html, 不要保存成了main.htm或任何其它不正确的文件名；

 

b.      _files (文件夹)

此文件夹中存放除主文件中的一切文件，例如main.html中嵌入图片的源文件等，都放在这个文件夹中。

 

3.       页面布局规范

由于邮件客户端对基于<div>和CSS的布局支持不理想，所以信纸模板页面的布局采用表格法。一般地，信纸是这样一块区域：其上、下、左、右有起装饰作用的图片，中间有一块干净区域，用于正文，为了达到这样的效果，在此推荐2种布局选择：

布局1

布局2

其中： 页眉装饰图区  页脚装饰图区  左装饰图区  右装饰图区  邮件正文区

这2种布局方式都是“表格套表格”的布局方式。为了达到效果的和谐，尺寸上需要注意以下几点：

全局：         width() = width()

                        height() = height()

                        这两个是显而易见的条件，而在不同布局方案下，还有一些更复杂的规则需要注意：

布局1：      width() + width() + width() = width() = width()

布局2：      height() + height() + height() = height() = height()

                        想一下的话，这个也不是很复杂。

这一部分规范内容的适用者包括：页面全图的设计者(PSD大图制作者)、剪裁者(PSD大图切分为页面中嵌入的散图的制作者)和页面编码者(HTML页面设计者)。

 

4.       HTML编码规范

编辑main.html时，您可能会使用Dreamweaver等使用工具。为了最终邮件展示效果的正确性，在编辑最终完成前，请检查您的文档是否满足以下几条原则：

a.       邮件发送系统全局采用GB2312编码，如果您的当前编码不是GB2312，请进行修改，修改方法是在修改<head>标签中

<meta http-equiv="Content-Type" content="text/html; charset=[xxx]" />

一行，将[xxx]替换为gb2312，如下所示：

如果没有这一行，就请在<head>标签中添加如下一行代码：

<meta http-equiv="Content-Type" content="text/html; charset=gb2312" />

 

b.      删除<head>标签中的<title>标签，否则页面标题将出现在邮件正文中；

 

c.       不要使用背景图片

无论是通过background属性，还是通过CSS的background:url()属性定义的背景图片，都不能保证在所有地方都正确显示(Outlook, Thunderbird中无法显示)，在邮件中使用图片，万无一失的方法就是使用<img>标签进行嵌入。可使用<table>进行总体排版布局，更详细的规范将在下面定义：

 

d.      尽量不要使用JavaScript

出于安全性的考虑，绝大多数邮件端都屏蔽了JavaScript，因此除非万不得已，不要在模板中添加JavaScript

 

e.      尽量少使用CSS

由于对于HTML电子邮件的CSS支持目前还缺乏国际标准，不同邮件端对于CSS的支持均不尽相同，为了达到最好的展现效果，做到最大兼容，因此建议：能够不使用CSS的地方，就不要使用CSS，而改用传统的HTML属性进行样式定义。例如，对于表格背景色的定义，使用

 

<table bgcolor=”#ffffff”>

比使用

<table style=”background:#ffffff”>

 

兼容性更好；以下是不推荐使用CSS的几个常用属性：
